新版picgo之piclist的兰空图床篇

新版picgo之piclist的兰空图床篇

新版picgo

其实说新版也不合适，就是在使用picgo的过程中发现很多不方便的地方，主要就是不能同步删除云端图片这项，然后就网上找呗。

是基于picgo二次开发的项目，改了名叫piclist

直接下载安装就好了，如果你已经安装了picgo，这个里面有一键迁移功能，很方便。

兰空图床的配置

兰空图床需要您自行搭建服务端，详情请参考兰空图床

piclist的文档说明里有怎么配置的，但是对于小白来说可能有点迷糊吧。

在配置中的难点主要在于如何获取兰空图床V2版本的tocken

  • 登录到一个 V2 版本的兰空图床，左侧菜单栏点击进入 API 接口页面 查看获取一个 token 的方式：

  • 使用 postman 之类的 api 调试工具发起一个 http post 请求即可生成一个 token，请求时的细节如下

    1. 请求 url（就是上面那个接口URL）: https://你自己的地址/api/v1/tokens​

    2. 请求方法: POST

    3. 设置请求头 Accept​ 的值为 application/json​

    4. 请求体中使用 json 语法填入邮箱和密码：

    5. 请求成功后得到返回信息中的 token​

    6. 使用 Bearer​ 拼接拿到的 token 作为 Auth token​ 填入 PicGo 设置中。注意 Bearer 和 返回的 token 之间有个空格，请严格按照格式填写，程序不会校验这个细节，不过这里有一点需要注意的是，如果配置后上传图片不成功，可多试几次，换几次tocken。

  • 另外需要注意的是，上传压缩图片，如果在piclist里设置了，兰空图床里最好不要设置压缩，不然二次压缩后图片太难看啦。
